Contributing to MechanicsDSL

Thank you for your interest in contributing! Here's how you can help:

Reporting Bugs

- Use GitHub Issues
- Include: Python version, OS, error message, minimal example
- Search existing issues first

## Suggesting Features

- Open an issue with "Feature Request" label
- Explain use case and benefits
- Provide example of desired syntax/behavior

## Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature-name`
3. Make your changes
4. Add tests for new functionality
5. Ensure all tests pass: `pytest tests/`
6. Update documentation
7. Commit with clear messages
8. Push and open a PR

## Code Style

- Follow PEP 8
- Use type hints where appropriate
- Add docstrings for public functions
- Keep functions focused and < 50 lines when possible

## Testing

- Write tests for all new features
- Maintain >90% code coverage
- Test edge cases

## Documentation

- Update README.md for new features
- Add examples to `examples/` directory
- Document DSL syntax extensions
